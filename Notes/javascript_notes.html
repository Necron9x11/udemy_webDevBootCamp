Scope

variables defined inside a fuHigh Order Functions on are not accessible outside the function variables defined outside a function are available inside a function. \

Think of it like Chinese Dolls where anything defined in a lower container has access to everythin gin the higher containers. Child scopes have access to objects defined inside a Parent scope.

So to be safe if you use a var inside a function always declare it so the scope is local tot he function. That way if the same var exists outside the function, in the global scope, you will not trample on it and cause yourself grief.

Two types of Object notation:

  • Bracket Notation
    • console.log(person["name"]);
    • Can be Used to Lookup a Variable
      • var str = "name";
      • someObject.str; //doesn't look for "name"
      • someObject[str]; //does evaluate str snd looks for name
  • Dot Notation
    • console.log(person.name)
    • can't be used if property starts with a number
      • someObject.4blah; //INVALID
      • someObject["4blah"]; //VALID
    • You cannot use dot notation for property names with spaces
      • someObject.fav color; //INVALID
      • someObject["fav color"] //VALID;
